Веб-сервіс DMMessageService

Цей веб-сервіс використовується для асинхронної інтеграції Документообігу з іншими додатками та конфігураціями за принципом "відправив і забув". Додаток-споживач відправляє в Документообіг запити на обробку шляхом виклику операції put і отримує відповіді викликом операції get. Обробка повідомлень може виконуватися Документообігом в будь-якому порядку. На один запит може бути сформовано кілька відповідей. Тому цей веб-сервіс зручно використовувати для інтеграції «1С:Документообіг 8 КОРП для України» з іншими програмами на платформі ESB (Enterprise Service Bus).

Операція: get

Значення, що повертається: DMMessage

Параметр Тип Опис
queue string Найменування черги повідомлень, які є вхідними по відношенню до інтегрованого з 1С:Документообігом додатком. До виклику цієї операції чергу повинна бути створена і налаштована 1С:Документообіг адміністратором. Детальніше про налаштування черг розповідається в описі конфігурації 1С:Документообіг.

Извлекает одно новое сообщение из указанной в параметре Queue очереди и возвращает его как объект класса DMMessage. Если новых сообщений нет, то возвращается объект класса DMOK. Если при выполнении этой операции произошла ошибка, например, неверно указана очередь, то возвращается объект класса DMERROR. Приклад виклику цієї операції в 1С:Підприємстві: 

МестоположениеWSDL = АдресПубликацииВебСервиса + "ws/dm.1cws?wsdl";

Определение = Новый WSОпределения(
	МестоположениеWSDL, 
	ИмяПользователя,
	Пароль);

Прокси = Новый WSПрокси(
	Определение,
	"http://www.1c.ru/dm",
	"DMService",
	"DMServiceSoap");
	
Прокси.Пользователь = ИмяПользователя;
Прокси.Пароль = Пароль ;

Ответ = Прокси.get("Incoming");
	
Если Ответ.Тип() = Прокси.ФабрикаXDTO.Тип("http://www.1c.ru/dm", "DMOK") Тогда
	// Немає відповідей для обробки
КонецЕсли;

Значення, що повертається: DMResponse

Параметр Тип Опис

Поміщає зазначене повідомлення в чергу повідомлень для подальшої обробки. Повертає DMOK у разі успішного приміщення запиту в чергу. Приклад використання цієї операції в 1С:Підприємстві:

Веб-сервіс DMService

Цей веб-сервіс використовується для синхронної інтеграції Документообігу з різними конфігураціями і додатками. Синхронна інтеграція застосовується для реалізації інтерфейсу користувача і для неинтерактивного онлайн взаємодії інтегрованих конфігурацій і додатків. Приклади використання цього веб-сервісу можна знайти в конфігурації Бібліотека інтеграції з 1С:Документообігом (БІД). Ця підсистема включає в себе набір команд, форм і модулів, необхідних для вбудовування користувальницького інтерфейсу Документообігу в інші конфігурації.
